# 📘 חשיבות הרווחים – Indentation - הזחה

## ✨ הסבר
בפייתון, **הרווחים** הם לא רק עניין של עיצוב — הם חלק מהתחביר עצמו.  
הם אומרים למחשב *מה שייך למה*.  

אם לא תכניסו רווחים נכונים, הקוד פשוט לא יעבוד.  
במילים אחרות – **הרווחים הם כמו סוגריים גדולות ל**

הם ישמשו אותנו בתנאים, בלולאות בפונקציות ובקלאסים

## 🔍 הסברים ודוגמאות

### דוגמה תקינה
```python
if 5 > 2:
    print("Five is greater than two")
````

* הפונקציה `print()` נמצאת תחת התנאי `if` ולכן היא מוזחת פנימה (בדרך כלל 4 רווחים).
* המשמעות: רק אם התנאי מתקיים – הפקודה תופעל.

### טעות נפוצה – בלי רווח

```python
# This will cause an error
# if 5 > 2:
# print("This line is not indented")
```

❌ כאן השורה השנייה לא מוזחת → פייתון לא יודע שהפקודה שייכת ל־`if` ולכן תופיע שגיאה.

## 💡 טיפים מתקדמים

* השתמשו תמיד **באותו מספר רווחים** (מומלץ 4 רווחים).
* אל תערבבו טאב ורווחים – זה עלול לשבור את הקוד.
* רוב העורכים (כמו VS Code או PyCharm) מוסיפים הזחה אוטומטית – נצלו את זה!

## 📍 היכן נשתמש ברווחים (Indentation) בפועל

הרווחים קובעים **לאיזה בלוק קוד** שורות משתייכות.
כל מקום שבו יש **בלוק פתוח (כמו תנאי, לולאה, פונקציה וכו׳)** – נשתמש בהזחה.

### 🧩 1. תנאים (if / elif / else)

```python
if grade >= 90:
    print("מצוין!")
elif grade >= 80:
    print("טוב מאוד")
else:
    print("תשתפר בפעם הבאה")
```

* כל שורה שמתבצעת *רק אם* התנאי מתקיים — חייבת להיות מוזחת.
* אפשר לקנן תנאים בתוך תנאים, וכל רמה מוסיפה עוד הזחה.

### 🔁 2. לולאות (for / while)

```python
for i in range(5):
    print(i)
    if i == 3:
        print("הגענו לשלוש!")

while count < 10:
    count += 1
    print("מספר:", count)
```

* כל מה שנמצא בתוך הלולאה מוזח.
* אם יש תנאי בתוך לולאה – יהיו כמה רמות של רווחים.

### ⚙️ 3. פונקציות (def)

```python
def greet(name):
    print("Hello", name)
    if name == "Dvir":
        print("Welcome back!")
```

* גוף הפונקציה תמיד מוזח.
* גם בפנים, תנאים ולולאות מקבלים הזחה נוספת.

### 🧱 4. מחלקות (class)

```python
class Student:
    def __init__(self, name):
        self.name = name

    def introduce(self):
        print("My name is", self.name)
```

* כל פונקציה בתוך מחלקה מוזחת.
* בתוך הפונקציה עצמה – הזחה נוספת (כל רמה מוסיפה טאב נוסף).

### 🎯 5. חריגות (try / except / finally)

```python
try:
    x = int(input("Enter a number: "))
    print("Success!")
except ValueError:
    print("That's not a number!")
finally:
    print("Done.")
```

* כל בלוק `try`, `except`, `finally` דורש רווחים פנימיים.
* אחרת – פייתון לא ידע אילו פקודות שייכות לכל בלוק.

### 🧠 6. מבנים מקוננים

```python
for i in range(3):
    if i == 1:
        print("נמצא התאמה!")
```

* כל רמה של מבנה מוסיפה הזחה אחת נוספת.
* קל לראות את ההיררכיה רק לפי הרווחים – בלי צורך בסוגריים!
