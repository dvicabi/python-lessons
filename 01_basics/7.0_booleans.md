# 📘 בוליאנים, השוואות ואופרטורים לוגיים - Booleans & Logic

## ✨ הסבר

בוליאן הוא סוג נתון שיכול להיות רק אחד משני ערכים:

```python
True
False
````

כלומר:

* `True` = אמת
* `False` = שקר

אפשר לחשוב על זה כמו מתג של אור:

```text
או שהאור דולק
או שהאור כבוי
```

אין מצב שלישי.

בעזרת בוליאנים אנחנו יכולים לשאול את המחשב שאלות, והמחשב מחזיר לנו תשובה של אמת או שקר.

לדוגמה:

```python
print(5 > 3)
```

הפלט יהיה:

```text
True
```

כי באמת 5 גדול מ־3.

### הלמה

בוליאנים הם הבסיס של קבלת החלטות בתכנות. בלי אמת ושקר, אי אפשר לבנות תנאים, בדיקות, משחקים, מערכות התחברות או תוכנות שמגיבות לפי מצב מסוים.

---

## 🔍 יצירת משתנים בוליאניים

אפשר לשמור ערך בוליאני בתוך משתנה:

```python
is_sunny = True
is_raining = False
```

אפשר גם להדפיס אותם:

```python
print(is_sunny)
print(is_raining)
```

הפלט:

```text
True
False
```

חשוב מאוד:

```python
True
False
```

נכתבים עם אות גדולה בהתחלה.

לא נכון:

```python
true
false
```

כי פייתון תחשוב שמדובר בשמות של משתנים.

---

## 🔍 השוואות מחזירות בוליאן

כאשר אנחנו משווים בין ערכים, פייתון מחזירה לנו תשובה בוליאנית.

כלומר, התוצאה של השוואה היא תמיד:

```python
True
```

או:

```python
False
```

דוגמאות:

```python
print(5 > 3)
print(10 == 2)
print(7 < 5)
```

הפלט:

```text
True
False
False
```

---

## 🔍 אופרטורים להשוואה - Comparison Operators

אופרטורים להשוואה הם סימנים שמאפשרים לנו לבדוק קשר בין ערכים.

| אופרטור | משמעות         | דוגמה    |
| ------- | -------------- | -------- |
| `==`    | שווה ל         | `5 == 5` |
| `!=`    | לא שווה ל      | `5 != 3` |
| `>`     | גדול מ         | `5 > 3`  |
| `<`     | קטן מ          | `3 < 5`  |
| `>=`    | גדול או שווה ל | `5 >= 5` |
| `<=`    | קטן או שווה ל  | `4 <= 5` |

דוגמה בקוד:

```python
a = 10
b = 3

print(a == b)
print(a != b)
print(a > b)
print(a < b)
print(a >= b)
print(a <= b)
```

הפלט:

```text
False
True
True
False
True
False
```

### הלמה

אופרטורים להשוואה מאפשרים לנו לבדוק מצבים. למשל: האם תלמיד עבר מבחן, האם משתמש הכניס סיסמה נכונה, האם מספר גדול ממספר אחר.

---

## ⚠️ ההבדל בין `=` לבין `==`

זו אחת הטעויות הכי נפוצות אצל מתחילים.

### `=` משמש להצבה

```python
age = 25
```

המשמעות:

```text
שמור את הערך 25 בתוך המשתנה age
```

### `==` משמש להשוואה

```python
print(age == 25)
```

המשמעות:

```text
האם age שווה ל־25?
```

דוגמה מלאה:

```python
age = 25

print(age == 25)
print(age == 30)
```

הפלט:

```text
True
False
```

### הלמה

`=` נותן ערך למשתנה.
`==` בודק האם שני דברים שווים.
אם מתבלבלים ביניהם, הקוד עלול לא לעבוד או לתת תוצאה לא נכונה.

---

## 🔍 השוואת מחרוזות

אפשר להשוות גם טקסטים.

```python
name = "Dvir"

print(name == "Dvir")
print(name != "Avi")
```

הפלט:

```text
True
True
```

אפשר גם לבדוק אם שני טקסטים שונים:

```python
print("apple" == "Apple")
```

הפלט:

```text
False
```

למה?

כי פייתון רגישה לאותיות גדולות וקטנות.

```text
apple
Apple
```

הם לא אותו טקסט.

### הלמה

השוואת מחרוזות חשובה מאוד כאשר בודקים שמות משתמש, סיסמאות, תשובות של תלמידים, פקודות שהמשתמש הקליד ועוד.

---

## 🔍 אופרטורים לוגיים - Logical Operators

לפעמים אנחנו רוצים לבדוק יותר מתנאי אחד.

בשביל זה משתמשים באופרטורים לוגיים:

```python
and
or
not
```

---

## 🔍 האופרטור `and`

`and` אומר: גם התנאי הראשון וגם התנאי השני חייבים להיות נכונים.

דוגמה:

```python
age = 20
has_ticket = True

print(age >= 18 and has_ticket == True)
```

הפלט:

```text
True
```

כי:

* `age >= 18` הוא `True`
* `has_ticket == True` הוא `True`

ולכן כל הביטוי הוא `True`.

### דוגמה נוספת

```python
age = 16
has_ticket = True

print(age >= 18 and has_ticket == True)
```

הפלט:

```text
False
```

כי גיל 16 לא עומד בתנאי הראשון.

### הלמה

משתמשים ב־`and` כאשר צריך שכל התנאים יתקיימו ביחד.

---

## 🔍 האופרטור `or`

`or` אומר: מספיק שאחד מהתנאים יהיה נכון.

דוגמה:

```python
is_teacher = True
is_admin = False

print(is_teacher or is_admin)
```

הפלט:

```text
True
```

כי לפחות אחד מהם נכון.

דוגמה נוספת:

```python
is_teacher = False
is_admin = False

print(is_teacher or is_admin)
```

הפלט:

```text
False
```

כי אף אחד מהתנאים לא נכון.

### הלמה

משתמשים ב־`or` כאשר מספיק שתנאי אחד יתקיים כדי שהתוצאה תהיה אמת.

---

## 🔍 האופרטור `not`

`not` הופך את הערך הבוליאני.

כלומר:

```python
not True
```

יהפוך ל:

```python
False
```

ו־

```python
not False
```

יהפוך ל:

```python
True
```

דוגמה:

```python
is_raining = False

print(not is_raining)
```

הפלט:

```text
True
```

כלומר, אם לא יורד גשם, אפשר לצאת החוצה.

### הלמה

משתמשים ב־`not` כדי לבדוק מצב הפוך. למשל: אם המשתמש לא מחובר, אם המשימה לא הושלמה, אם אין שגיאה.

---

## 🔍 דוגמה מאוחדת

```python
age = 18
grade = 85
is_student = True

print("Age is 18:", age == 18)
print("Grade is above 70:", grade > 70)
print("Is student:", is_student)

print("Can enter course:", age >= 18 and grade >= 70)
print("Needs help:", grade < 60 or is_student == False)
print("Is not student:", not is_student)
```

הפלט:

```text
Age is 18: True
Grade is above 70: True
Is student: True
Can enter course: True
Needs help: False
Is not student: False
```

בדוגמה הזאת השתמשנו ב:

* משתנה בוליאני
* השוואות
* `and`
* `or`
* `not`
* הדפסות של תוצאות אמת ושקר

---

## 🔍 שימוש בבוליאנים בתוך תנאי

בוליאנים חשובים במיוחד כאשר נגיע לתנאים.

דוגמה פשוטה:

```python
is_hungry = True

if is_hungry:
    print("Time to eat!")
```

הפלט:

```text
Time to eat!
```

המשמעות:

```text
אם is_hungry הוא True, תדפיס Time to eat!
```

דוגמה עם השוואה:

```python
grade = 90

if grade >= 55:
    print("Passed")
```

הפלט:

```text
Passed
```

### הלמה

תנאי `if` עובד לפי אמת או שקר. לכן הבנה של בוליאנים והשוואות היא הכנה ישירה לשיעור תנאים.

---

## 🔍 bool()

אפשר להשתמש ב־`bool()` כדי לבדוק איך פייתון מתייחסת לערכים מסוימים כאמת או שקר.

```python
print(bool(0))
print(bool(1))
print(bool(""))
print(bool("hi"))
```

הפלט:

```text
False
True
False
True
```

כללים פשוטים:

| ערך                | התוצאה של `bool()` |
| ------------------ | ------------------ |
| `0`                | `False`            |
| `1`                | `True`             |
| `""` מחרוזת ריקה   | `False`            |
| `"hi"` טקסט לא ריק | `True`             |

### הלמה

זה עוזר להבין איך פייתון מחליטה אם ערך מסוים נחשב "קיים" או "ריק".

---

## ⚠️ טעויות נפוצות

### טעות 1: לכתוב true במקום True

לא נכון:

```python
is_sunny = true
```

נכון:

```python
is_sunny = True
```

---

### טעות 2: להתבלבל בין `=` לבין `==`

לא נכון לצורך השוואה:

```python
age = 18
print(age = 18)
```

נכון:

```python
age = 18
print(age == 18)
```

---

### טעות 3: לחשוב שאות גדולה ואות קטנה הן אותו דבר

```python
print("apple" == "Apple")
```

הפלט:

```text
False
```

---

### טעות 4: להשתמש ב־and כשצריך or

```python
day = "Friday"

print(day == "Friday" and day == "Saturday")
```

הפלט:

```text
False
```

למה?

כי יום אחד לא יכול להיות גם Friday וגם Saturday באותו זמן.

במקרה כזה צריך:

```python
day = "Friday"

print(day == "Friday" or day == "Saturday")
```

הפלט:

```text
True
```

---

## 💡 טיפים מתקדמים

* `True` ו־`False` חייבים להיכתב עם אות גדולה בהתחלה.
* כל ביטוי השוואה מחזיר בוליאן.
* `==` בודק שוויון.
* `=` מבצע הצבה.
* `and` דורש שכל התנאים יהיו נכונים.
* `or` דורש שלפחות תנאי אחד יהיה נכון.
* `not` הופך את התוצאה.
* פייתון רגישה לאותיות גדולות וקטנות גם במחרוזות וגם בשמות משתנים.

## 🧠 סיכום השיעור

בשיעור הזה למדנו שבוליאן הוא סוג נתון שיכול להיות רק `True` או `False`.

למדנו להשתמש באופרטורים להשוואה:

* `==` שווה ל
* `!=` לא שווה ל
* `>` גדול מ
* `<` קטן מ
* `>=` גדול או שווה ל
* `<=` קטן או שווה ל

בנוסף למדנו להשתמש באופרטורים לוגיים:

* `and` כאשר כל התנאים צריכים להתקיים
* `or` כאשר מספיק שתנאי אחד יתקיים
* `not` כדי להפוך את התוצאה

ראינו גם שבוליאנים משתלבים בצורה טבעית עם תנאים, ולכן הם שלב חשוב מאוד לפני לימוד `if`.

### הלמה הגדול של השיעור

בוליאנים הם הדרך של המחשב לענות על שאלות של כן או לא.

ברגע שאנחנו יודעים לבדוק האם משהו נכון או לא נכון, אנחנו יכולים להתחיל לבנות תוכנות שמקבלות החלטות.
